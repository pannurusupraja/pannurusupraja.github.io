RESCHEDULE FLOW

Reschedule Flow Ezeeinfo Bitla TY BOGDS

Ezeeinfo Reschedule Flow

Rechedule Policy will be real time call,once vendor enabled will reflect in integration layer

Vendor API:

[Domain-name]/busservices/api/3.0/json/[operatorCode]/[username]/[api-token]/search/[fromCity]/[toCity]/[tripDate]

Reschedule parameters:

    ticketTransferTerms.transferable
    ticketTransferTerms.allowedTill

If transferable is ‘1’ reschedule will enable

Else reschedule won’t allow

Bitla Reschedule Flow

Reschedule policy will take from operators proxy,proxy will update every day once.

[DomainName]/bitlaNew/Operator/[Operator-Id].json

If Allow_reschedule is true from avroutes

    will construct reschedule policy from above api

Else won’t allow to reschedule

Reschedule parameter:

Reschedule_policies

TY Reschedule Flow

Reschedule policy will be the part of routedetails,so vendor enables the policy will update in integration

Vendor API:

[DomainName]/APISearchRoutesV2?AgentID=[AgentID]&KeyCode=[KeyCode]&

FromCityID=[FromCityID]&ToCityID=[ToCityID]&JourneyDate=[JourneyDate]

Reschedule Parameters:

    IsRescheduleAllowed
    ModificationTill

If IsRescheduleAllowed is true

    Reschedule will Enabled

Else won’t allow to reschedule

Reschedule Flow

Vendor will enable the reschedule policy based on route level or operator level.

Redbus will enable based on route level => If Not enabled Need to check below cases

    vendor enabled or not.
    If Enabled, by realtime update it will reflect on front end.

Based on parameter check will enable the reschedule policy in our end

Once operator enables the reschedule policy, on realtime update it will reflect in Front end.
